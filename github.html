Criando uma Conta e Configurando o GitHub

Como criar uma conta no GitHub?

O primeiro a se fazer é entrar no site do GitHub clicando aqui. Após isso, você será redirecionado para o site do GitHub, e no canto superior direito terá duas opções: Sign In (para quem já tem uma conta) e Sign Up (para cadastrar uma nova conta).

Quais são os primeiros passos após a criação da conta no GitHub?

  • Configure seu perfil para que fique apresentável, existe um site com textos personalizados para usar no seu perfil: MarkdownBadges.
  • Preencha as informações de biografia e crie um README (Crie um arquivo com o nome README.MD).

5. Como configurar o Git no seu computador local para se conectar ao GitHub?

Passo 1: Instalar o Git. Se você ainda não tiver o Git instalado, baixe e instale a versão apropriada para o seu sistema operacional a partir do site oficial do Git: https://git-scm.com/downloads.

Passo 2: Configurar suas informações de usuário. Abra o terminal ou prompt de comando e configure seu nome de usuário e endereço de e-mail com os seguintes comandos:

git config --global user.name "Seu Nome"
git config --global user.email "seu-email@example.com"

Repositórios

O que é um repositório no GitHub?

Um repositório no GitHub é um meio de armazenar projetos e dados seus em um lugar seguro.

Como criar um novo repositório no GitHub?

Vá para a página Home do GitHub e clique no botão "New repository". Dê um nome ao seu repositório e selecione se será público ou privado.

Qual é a diferença entre um repositório público e um repositório privado?

Um repositório público pode ser visualizado e baixado por qualquer pessoa, enquanto um repositório privado só pode ser acessado por você e por quem você autorizar.

Branches e Merge

O que são branches e como criar uma nova branch no GitHub?

Branches são ramificações que permitem que você divida seu trabalho em partes separadas. Para criar uma nova branch, use:

git checkout -b nome-da-branch

No GitHub:

  • Vá para a página do seu repositório.
  • Clique no menu suspenso de branches.
  • Digite o nome da nova branch e clique em "Create branch".

Como fazer merge de branches e quais são as boas práticas para isso?

Para fazer merge, use:

git checkout main git merge nome-da-branch

No GitHub, vá para a página do pull request da branch e clique em "Merge pull request".

Boas práticas:

  • Revise o código antes de fazer o merge.
  • Certifique-se de que todos os testes estão passando.
  • Resolva conflitos de merge.
  • Use rebase para manter um histórico limpo quando apropriado.

Contribuições e Pull Requests

Como fazer um fork de um repositório?

Um fork é uma cópia de um repositório que você pode modificar. Para fazer um fork:

  • Vá para o repositório que deseja forkar.
  • Clique no botão "Fork".
  • Selecione sua conta pessoal ou uma organização para copiar o repositório.

O que é um pull request e como criar um pull request?

Um pull request é uma solicitação para mesclar mudanças de uma branch em um repositório de destino. Para criar um pull request:

  • Faça push das mudanças para sua branch:
    • git push origin nome-da-branch
  • No GitHub, vá para a página do repositório e clique em "Compare & pull request".
  • Preencha o título e a descrição do pull request e clique em "Create pull request".

Como revisar e aceitar um pull request?

Para revisar um pull request:

  • Vá para a aba "Pull requests" no repositório.
  • Clique no pull request que deseja revisar.
  • Veja os arquivos alterados e adicione comentários, se necessário.

Para aceitar um pull request:

  • Se estiver satisfeito com as mudanças, clique em "Merge pull request".
  • Confirme o merge clicando em "Confirm merge".

Issues e Projetos

18. O que são issues e como abrir uma issue no GitHub?

Issues são relatórios de bugs, solicitações de recursos ou outras tarefas. Para abrir uma issue:

  • Vá para a aba "Issues" no repositório.
  • Clique em "New issue".
  • Preencha o título e a descrição da issue e clique em "Submit new issue".

19. Como usar o sistema de projetos do GitHub para gerenciar tarefas?

Projetos no GitHub ajudam a organizar e priorizar o trabalho. Para criar um projeto:

  • Vá para a aba "Projects" no repositório.
  • Clique em "New project".
  • Dê um nome e descrição ao projeto e adicione colunas como "To do", "In progress", "Done".
  • Adicione cartões para issues, pull requests ou notas.
